# Q3 — AI Agent: GCS Bucket Setup (12.5 marks)

Prompt your **AI coding agent** (Claude Code, Cursor, Copilot CLI, etc.) to set
up Google Cloud CLI access, enable Cloud Storage, and create your **assigned**
bucket in `asia-south1` — then make it public so the server can verify it.

**Marks:** 6.25 auto-graded (server checks the bucket via public GCS metadata)
+ 6.25 graded offline from the **agent log** you paste.

## Step 0 — Read your task details file

Download "Q1 task details" from the question. It contains your **assigned
bucket name** (e.g. `q1-<nonce>`). The bucket name must match **exactly** —
the grader literally checks that this name appears in your log and exists
publicly. If the question ever updates its naming scheme, re-download the
details file and use the new name.

## Step 1 — One-time GCP setup (manual, before the agent runs)

- A Google account with a **billing-enabled** project (free trial works).
- Install the gcloud CLI, then `gcloud auth login` and
  `gcloud config set project <your-project>`.
- Practice the flow once by hand (the question itself advises this).

## Step 2 — Prompt the agent (single session, single prompt)

Give your agent one prompt like:

> Using the gcloud CLI (already authenticated), enable the Cloud Storage API,
> create the bucket `q1-<your-nonce>` in `asia-south1` with uniform
> bucket-level access, make it publicly readable AND listable (allUsers →
> objectViewer and legacyBucketReader), then verify it anonymously via the
> public GCS JSON API. Log every step you take as JSONL.

The commands the agent should end up running:

```bash
gcloud services enable storage.googleapis.com
gcloud storage buckets create gs://q1-<nonce> --location=asia-south1 --uniform-bucket-level-access
gcloud storage buckets add-iam-policy-binding gs://q1-<nonce> --member=allUsers --role=roles/storage.objectViewer
gcloud storage buckets add-iam-policy-binding gs://q1-<nonce> --member=allUsers --role=roles/storage.legacyBucketReader
gcloud storage buckets describe gs://q1-<nonce> --format=json
gcloud storage ls gs://q1-<nonce>
```

**Both** IAM roles matter: `objectViewer` makes objects readable,
`legacyBucketReader` makes the bucket **listable** — the question asks for
readable *and* listable.

## Step 3 — Verify like the grader does (anonymously)

```bash
curl https://storage.googleapis.com/storage/v1/b/q1-<nonce>
```

No auth. You should get JSON with `"location": "ASIA-SOUTH1"`. If this works
from your machine without credentials, the auto-grader will pass you.

## Step 4 — The agent log (the other 6.25 marks)

Paste a **JSONL** log (one JSON object per line) showing the agent's run. It
must contain, at minimum:

- the **exact assigned bucket name** (the validator string-matches it!)
- Storage **API enablement**
- bucket **creation** (with location)
- the public-access IAM changes
- **describe / list output**

A clean shape per line:

```json
{"timestamp": "...", "actor": "ai-coding-agent", "step": "create_bucket", "command": "gcloud storage buckets create ...", "output": "...", "status": "success"}
```

Have the agent write this log itself as it works — that's the whole point of
the task ("all task executions must be done solely by your agent").

## Gotchas that cost people marks

- **"Agent log does not contain the expected bucket name"** → your bucket name
  doesn't match the task-details file. Re-download it; the assignment may have
  changed since you first read it.
- Bucket in the wrong region → auto-check fails. It must be `asia-south1`.
- Only `objectViewer` granted → bucket not listable → verification can fail.
- Deleting the bucket after Check → grading happens **after the deadline**;
  keep the bucket (an empty bucket costs ~₹0/month).
- New GCP projects sometimes can't link billing (quota) → host the bucket in
  any billing-enabled project you already have; the grader only checks the
  bucket, which is globally named.

## Checklist

- [ ] `asia-south1`, uniform access, allUsers = objectViewer + legacyBucketReader
- [ ] Anonymous `curl` of the bucket metadata works
- [ ] JSONL log pasted: bucket name + API enable + create + IAM + describe/list
- [ ] Check shows "GCP Cloud bucket verified. Auto-graded score: 6.25/12.5" —
      **that IS the full visible score**; the other 6.25 comes after the deadline
- [ ] Saved on SEEK; bucket left alive until results are out
