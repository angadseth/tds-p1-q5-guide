# Q4 — AI Agent: Dataset Upload to GCS (12.5 marks)

Prompt your AI coding agent to take the **server-generated** `eval.jsonl`
(51 rows: 17 positive / 17 negative / 17 neutral reviews), upload it
**unchanged** to your assigned Q2 bucket, and compute its **SHA-256**.

**Marks:** 6.25 auto (server verifies bucket, object metadata, content, and
hash) + 6.25 offline from your pasted agent log.

## Step 0 — Download YOUR files

From the question, download:

1. **Q2 task details** → contains your assigned bucket name (`q2-<nonce>`)
2. **eval.jsonl** → generated uniquely for your login

⚠️ **The two rules that matter most:**

- **Do NOT edit or regenerate eval.jsonl.** Not one byte. No re-saving in an
  editor that might change encoding/line-endings, no "cleaning". The server
  compares the SHA-256 of what's in your bucket against what it generated for
  you — any change = fail.
- **Re-download if the question updates.** If the task details or dataset get
  regenerated server-side, your old local copy will hash-mismatch. When in
  doubt, download fresh and re-upload.

## Step 1 — Prompt the agent

> The bucket `q2-<nonce>` must exist in `asia-south1` (create it exactly like
> Q3 if needed, public read+list). Upload the file `eval.jsonl` unchanged to
> the bucket as `eval.jsonl`, compute its SHA-256 with `sha256sum`, then
> verify by downloading it anonymously from the public URL and confirming the
> hash matches. Log every step as JSONL.

Commands the agent runs:

```bash
gcloud storage buckets create gs://q2-<nonce> --location=asia-south1 --uniform-bucket-level-access
gcloud storage buckets add-iam-policy-binding gs://q2-<nonce> --member=allUsers --role=roles/storage.objectViewer
gcloud storage buckets add-iam-policy-binding gs://q2-<nonce> --member=allUsers --role=roles/storage.legacyBucketReader

sha256sum eval.jsonl                      # local hash
gcloud storage cp eval.jsonl gs://q2-<nonce>/eval.jsonl
gcloud storage ls -l gs://q2-<nonce>
```

## Step 2 — The verification that actually proves it (do this!)

Download your own file **anonymously** — exactly what the grader does — and
compare hashes:

```bash
curl -s https://storage.googleapis.com/q2-<nonce>/eval.jsonl | sha256sum
```

If this hash equals your local `sha256sum eval.jsonl`, the auto-check will
pass. If it differs, something modified the file in transit (usually an
editor or a copy step that touched line endings).

## Step 3 — The agent log

Same JSONL format as Q3. Must include:

- the **exact assigned bucket name**
- bucket creation / public IAM steps
- the dataset inspection (51 rows, used **unchanged**)
- the local `sha256sum` output (the full hash)
- the upload command + result
- the anonymous public download + hash match

## Gotchas

- Object must be named exactly **`eval.jsonl`** at the bucket root.
- Uploading a re-saved copy (Notepad, Excel, pandas round-trip) silently
  changes bytes → hash mismatch → auto-fail. Upload the downloaded file as-is.
- "Agent log does not contain the expected bucket name" → bucket name changed
  server-side; re-download task details and redo with the new name.
- 6.25/12.5 after Check is the **maximum visible now** — the rest is offline.
- Keep the bucket + object alive until results are published.
